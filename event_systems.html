Event System

Overview

The event system consists of two parts: the event handler and the event triggers. Event triggers are instantiable classes that allow you to blindly trigger methods across your program, therefore allowing you to implement better staging within your program. The event handler is a static class that allows you to access these event triggers wherever and whenever in your program.

Event Handler

Get Event trigger

The static getEventTrigger(String) method takes in the reference name of the event trigger you want to use.

Add Event trigger

The static addEventTrigger(String, EventTrigger) method takes in the reference name of the event trigger and the event trigger object itself. However, this method is not meant to be used since event triggers automatically add themselves to the event handler on instantiation in their constructor.
