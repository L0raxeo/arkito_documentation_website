Transform

Overview

A Game Object’s transform encapsulates the positional attributes of its parent object.

Variable Hierarchy

  • Blue = class
  • Green = attributes

Z-Index

Since Arkito is a 2D game engine, multi-layer renderings must be sorted. An object with a higher z-index will be rendered on top of an object with a lower z-index. When the scene is updating its game objects, it will sort the order of its iterator from smallest z-index to largest z-index.

List of Methods

  • worldPosition()
  • getScreenPosition()
  • getScreenCenterPosition()
  • setWorldPosition(Vector2f)
  • setWorldPosition(float, float)
  • setWorldPosition(Vector3i)
  • setzIndex(int)
  • getzIndex()
  • move(Vector2f)

Transform Positions

A game object’s position is associated with the top left corner of its bounds. For example, when rendering an image texture, the left edge of the image will be the transform’s x-coordinate.

Get Screen Center Position

The center position of a transform refers to the center of its bounds.
Formula: position.x + (scale.x / 2); position.y + (scale.y / 2)
