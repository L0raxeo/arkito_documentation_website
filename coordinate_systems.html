Coordinate Systems

Overview

Arkito has different coordinate systems. For games, the world coordinate system is used because it adheres to the standard mathematical graph system while also accounting for the game camera’s position. Typically for mathematical calculations and GUI layouts, the graph coordinate system is used. Typically for mouse positions, post-render graphics, and GUI layouts the screen coordinate system is used.

Direction

Graph

  • x-coordinates increase towards the right of the application window
  • y-coordinates increase towards the top of the application window
Screen
  • x-coordinates increase towards the right of the application window
  • y-coordinates increase towards the bottom of the application window
World
  • x-coordinates increase towards the right of the application window, however, they are displaced according to the camera x-offset
  • y-coordinates increase towards the top of the application window, however, they are displaced according to the camera’s y-offset

Screen Position

Adds the Camera’s offset values to the raw screen position (Window_Height - worldPos.y)

Code Formula:
x = noCamScreenPosition..x + Camera.xOffset(); y = position.y + Camera.yOffset()
