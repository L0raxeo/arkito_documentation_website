Camera

Overview

The Camera is a static class that encapsulates the position offsets for world object rendering positions. When an object is rendered on the screen, it gets the screen position from the object’s transform component.

  • resetPosition()

  • getDegFromOriginToTarget(int, int, int, int)
  • screenToWorld(Vector2i)

  • xOffset()
  • yOffset()

Reset Position

The reset position function sets the xOffset and yOffset to zero.

Set World Position

The set world position function sets the xOffset to -worldPosition.x and yOffset to worldPosition.y as passed through the parameters.

Get World Position

Returns a vector with an x-value of -xOffset and yOffset.

Get Degree from Origin to Target

Returns the angle at which an object should be rotated in order to face the specified target position. Both screen positions are passed through the parameters. While this method is more of a mathematical utility function, it makes it easy and is often applicable to rotating player game objects towards a specific point (especially in top-down/bird’s-eye/helicopter view games).

Screen to World

Translates and returns the parameter’s screen position to world coordinates by subtracting the window height from the y-coordinate of the specified screen position.

World to Screen

Translates and returns the parameter’s screen position to world coordinates by subtracting the window height from the y-coordinate of the specified world position.

Move World Coordinates

Moves the Camera according to the velocities specified in the parameters. The vector passed through the parameters should be in world coordinates.

Offset Functions

Both functions return their respective offset values for their respective axes.
