Components

Overview

A component is stored in a list of components inherited from its respective game object. The abstract component class provides a blueprint for your own custom components. None of its methods are abstract, so it isnâ€™t necessary to override/implement any of them (this was done to reduce coupling). However, it was made an abstract class in order to prevent users from directly creating a component object that has no functionality. Component start, update, render, collision, and on-destroy methods are automatically invoked/handled by its parent game object. It is up to you to populate these methods with tasks.

UID System

Each component is identified with a unique identifier associated with the number of components created at the time of its instantiation.
