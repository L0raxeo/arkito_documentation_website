Animations

Overview

The animation object is an easy way to implement animation into your project. It’s a general object utility which means it can be applied to anything (Gui textures, component rendering… etc). When an animation object is instantiated, the frame interval (i.e. the speed of the animation) and the individual frames are passed through the parameters. If you want to update the animation, its update method needs to be invoked—usually through a component’s update method or through the scene’s update method. Rendering the animation is as simple as rendering the current frame’s buffered image through the graphics component in your render method (the current frame can be accessed via its getter method).

Frame Intervals

As the frame interval value increases so does the time between each frame. A lower frame interval means faster animation.

Methods

  • update()
  • getIndex()
  • getCurrentFrame()

Update

To animate the animation, invoke your animation object’s update method via an update method linked to the game thread (e.g. a scene’s update method or a component’s update method).

Get Current Frame

In order to render the animation, render the buffered image returned by the getCurrentFrame() method.

Get Index

Returns the index associated with the current frame of the animation.
